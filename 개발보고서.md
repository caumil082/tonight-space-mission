## 2. 기획 의도 (해결하려는 문제)

- 공공데이터포털의 **한국천문연구원\_천문현상 정보 API**는 글자 중심이라 중학생이 보기엔 딱딱하고, "동방최대이각·합삭" 같은 용어가 어렵다.
- **핵심 아이디어**: 공공데이터를 단순 "조회"가 아니라 **학생의 탐구 활동**으로 전환한다.
  - 어려운 용어 → 쉬운 말로 변환
  - "오늘 밤 볼 만한 것"을 카드·추천으로 즉시 제시
  - 미션·퀴즈·관측 일지로 직접 참여·기록 → **나만의 과학 탐구 포트폴리오**

## 3. 주요 기능

탭 구성: **오늘 · 달력 · 미션 · 일지 · 퀴즈 · 검색** (6탭)

| 기능 | 설명 |
|------|------|
| **오늘** | 🌟 대표 이벤트 히어로 카드(추천 점수 기반, 움직이는 이미지·D-day) + 오늘/다가오는 30일 현상 카드(접이식), ⭐ 관심(☆) 모음 |
| **달력** | 이번 달 캘린더(현상 있는 날에 점), 오늘/이번주/이번달 보기, 관심 등록일 ★ 표시 |
| **미션** | 현상별 관측 미션(보름달 스케치·별똥별 세기·행성 찾기·식 관찰). 결과(성공/일부/시도)가 **관측 일지로 자동 기록**(시도도 점수) |
| **일지** | 관측 기간(연·월·일·시), 하늘 상태, **시상·투명도·천구좌표**, 장소(GPS·구글지도), 사진(클라우드), 느낀 점 기록. 백업(JSON)·PDF 저장 |
| **퀴즈** | 행성·우주·별자리 **사진 맞추기**(매판 카테고리 골고루, 반복 회피). 성공 횟수 누적(목표 100), 하루 10회 제한 |
| **검색** | ① 위키백과(요약 + 관측 데이터·특성 인포박스 표) ② 딥스카이 목록(OpenNGC 1,721개, 이름/번호·별자리·등급 검색 + DSS 실사진) |

### 게임화 요소
- **XP·레벨(8단계)·레벨바**, **연속 스트릭(🔥)**, 미션 성공 시 점수 카운트업·꽃가루·레벨업 연출
- 관측 일지 기록이 곧 점수 → 꾸준히 기록하도록 유도

### 살아있는 화면
- 밤하늘 배경 애니메이션(반짝이는 별 + 별똥별), 히어로 카드의 현상별 움직이는 이미지(유성우·달·행성·식) + 좌상단 큰 달·우하단 염소자리·하단 걷는 염소

## 4. 기술 스택 / 시스템 구조

- **프런트엔드**: HTML5, CSS3, 바닐라 JavaScript (프레임워크·빌드 도구 없음)
- **저장**: 브라우저 `localStorage` (점수·일지·관심·퀴즈 기록) — 회원가입 없음
- **오프라인/캐시**: 서비스워커(`sw.js`, 네트워크 우선) → 온라인이면 항상 최신, 오프라인이면 캐시 동작
- **호스팅**: GitHub Pages (HTTPS)
- **이미지 클라우드**(선택): Firebase Storage(익명 인증, 보안 규칙)

데이터 흐름:
```
[공공데이터 천문현상 API] --(연 1회 수집 스크립트)--> data/events.js (앱에 번들, 오프라인)
[OpenNGC CSV] --(변환 스크립트)--> data/dso.js (앱에 번들)
[실행 시] 위키백과·DSS·Open-Meteo·Firebase = 인터넷 직접 호출(키 없음/공개)
```

## 6. 데이터·이미지 출처 및 라이선스

| 자료 | 출처 | 라이선스/이용 | 사용처 |
|------|------|----------------|--------|
| 천문현상 정보 | 한국천문연구원 (공공데이터포털) | 공공데이터 이용약관(출처표시) | 오늘·달력·미션·퀴즈의 현상 데이터 |
| 딥스카이 천체 목록 | OpenNGC | 자유 라이선스(재배포 가능) | 딥스카이 검색 |
| 천체 사진(퀴즈) | 위키미디어 공용 | 공개 자료 | 사진 맞추기 퀴즈 |
| 천체 정보 | 위키백과 REST/Action API | 공개 API | 검색-위키백과 |
| 딥스카이 실사진 | DSS (CDS hips2fits) | 공개 이미지 서비스 | 딥스카이 상세 |
| 현재 날씨 | Open-Meteo | 무료·키 없음 | 관측 기록 하늘 상태 자동 |
| 역지오코딩(지명) | BigDataCloud | 무료 클라이언트 API | 관측 장소 자동 |

> 모든 외부 API는 **API 키·결제 없이** 정적 클라이언트에서 직접 호출(또는 사전 번들). 코드(프로젝트)는 MIT 라이선스.

## 7. 보안 · 개인정보

- **비밀키 없음**: 코드/저장소에 비밀 키가 없다. (천문현상 API 키는 수집 시 1회만 사용, 코드 미포함)
- **XSS 방어**: 사용자 입력·외부 데이터는 모두 escape 처리. 사진은 `data:image`/`https` 형식만 허용(조작된 백업 차단). 공개 전 보안 점검에서 발견한 XSS 벡터 수정(v1.0.1).
- **개인정보 최소화**: 로그인·계정 없음. 관측 기록·사진은 기본적으로 사용자 기기에만 저장. 백업 파일(개인 기록)은 `.gitignore`로 공유 금지.
- **사진 클라우드(선택)**: Firebase Storage는 익명 인증 + "이미지·5MB 이하" 보안 규칙 적용. 천체 사진 용도(초상권 무관).
- **위치**: GPS는 사용자가 버튼을 눌러 권한을 허용할 때만 사용(HTTPS).

## 8. 개발 과정 (주요 마일스톤)

1. 기획서 작성 → 데이터 방식 결정(1년치 미리 받아 번들로 오프라인·비용 0)
2. v1 핵심 기능: 오늘·달력·미션·일지·퀴즈 + 추천 히어로
3. 실제 데이터 연결(천문현상 API 2026년 186건) + 용어/난이도 보강
4. 백업/복원(사진 포함), 관측 일지에 과거 기록·사진 추가
5. 전체 QA(여러 날짜·빈 상태·캐시·핵심 플로우) → git 저장소·태그(v1.0) → GitHub 공개(MIT)
6. 보안 점검(XSS 수정) → 공개 전환
7. 재미 업그레이드: 살아있는 배경·움직이는 히어로·성공 연출·접이식 카드·게임화(XP/레벨/스트릭)
8. 사진 맞추기 퀴즈(실사진), 관측 기록 고도화(기간·시상·투명도·천구좌표·GPS·날씨·구글지도)
9. 사진 클라우드(Firebase), 서비스워커(캐시 해결·오프라인), PDF 저장
10. 검색 탭(위키백과 인포박스 + OpenNGC 딥스카이 목록 + DSS 실사진)

## 10. 한계 및 향후 과제

- 천문현상 데이터는 **2026년**까지(2027년은 API 미공개). 연말쯤 재수집 예정.
- 날씨는 기상청 "공식"이 아닌 Open-Meteo(정적앱 CORS/키 제약). 공식 필요 시 서버리스 프록시 필요.
- 관측 장소는 구글지도 "링크"(핀 선택 임베드는 Google Maps 결제 필요).
- 딥스카이 목록은 ≤12등급+메시에(관측 가능 위주). 더 어두운 천체는 기준 상향 재생성 가능.
- 사진 클라우드·검색·DSS 등은 인터넷 필요(목록 검색 자체는 오프라인 가능).
- 향후: 날씨 연동 정확도, 관측 일지 통계/그래프, 학습 포트폴리오 내보내기, Tauri 데스크톱 앱.

## 11. 실행 · 배포 · 갱신 방법

- **실행**: `index.html`을 브라우저로 열거나, 공개 주소 접속(설치 불필요).
- **배포**: GitHub `main` 브랜치 → GitHub Pages 자동 게시.
- **천문현상 데이터 갱신**: `node tools/fetch-astro.mjs --key=공공데이터_Decoding키 --year=2027`
- **딥스카이 데이터 갱신**: OpenNGC `NGC.csv`를 받아 `node tools/gen-dso.mjs` 실행 → `data/dso.js` 재생성
